内网概述

内网（Intranet）是企业、机构或组织内部的私有网络，仅供授权用户使用，基于 TCP/IP 协议构建。它主要用于内部通信、资源共享和业务系统的访问，具备高度的安全性和隐私性。内网通过防火墙或 NAT 严格控制外部访问，并可通过 VPN、堡垒机或内网穿透实现安全的外部访问。其功能包括共享资源（如文件服务器、打印机、数据库）、统一用户管理（如域控制器和单点登录）、支持业务系统（如 ERP、CRM、OA）和团队协作（如即时消息、内部邮箱、视频会议）。此外，内网通过权限控制和防火墙确保资源的安全性，可有效防止外部攻击。

工作组

工作组是一组互相连接的计算机组成的一个对等（Peer-to-Peer）网络，没有中心服务器。每台计算机都是独立的，没有统一的管理者，用户可以自由加入或离开工作组，适用于小型网络设计，每台计算机需单独配置安全策略。工作组只需要计算器连接到同一个网络并加入同一个工作组即可实现资源共享，无需专用服务器或域控制器，但随着计算机的增加，管理账户将变得困难，且无集中式认证和管理，安全性较低。

工作组和域的关系

在计算机网络中，工作组（Workgroup） 和 域（Domain） 是两种不同的管理架构。它们的区别主要体现在身份验证和资源管理方式上。

域

域是在企业或组织内部网络中，用于管理和控制计算机、用户以及其他资源的逻辑分组和命名系统。它是由域控制器（DC）管理的一个网络实体，所有计算机和用户的账户信息都会在域控制器中进行集中管理，提供单点登录（SSO）功能，用户只需一次认证即可访问所有授权的资源，域通过统一的安全策略（如密码策略、权限控制等）来增强网络的安全性，只有域内的授权用户才能访问指定的资源。域通常使用DNS来命名和解析网络中的计算机和服务，确保设备能够通过域名访问资源，而不需要记住IP地址。内网域为组织提供了一种高效、安全和集中管理计算机、用户和网络资源的方法，确保内网的安全性和易管理性。

单域

单域（Single Domain）是指在一个组织内部网络中，所有计算机、用户和资源都属于同一个域，集中在同一个域控制器（DC）下进行管理的网络结构。在单域架构中，域控制器负责管理用户账户、计算机、资源和安全策略等，所有的认证、权限管理和访问控制都由该域控制器统一处理，一般在一个域内要建立至少两个域服务器，一个作为DC，一个是备份DC。单域结构适用于规模较小或管理简单的组织，通常只需要一个域控制器来管理所有内部资源。由于结构简单，配置和维护相对较容易，适合中小型企业或部门级网络。

父域

父域是指域树或域森林中的顶级域，它是域的层次结构中的上级部分。父域管理着一组子域，并为它们提供共享的安全策略和资源访问控制。在域结构中，父域的角色通常是对网络资源和身份进行集中管理和安全控制。父域控制整个网络的身份验证机制，确保只有授权的用户可以访问网络资源，为子域提供安全策略（如密码策略、登录策略等），子域可以继承这些策略，确保组织内的一致性和统一性，父域通常负责命名解析，它管理着包括子域在内的域名系统（DNS）。在大型公司中，父域可以用于管理多个部门的子域，简化跨部门的用户和计算机管理，对于跨地域的组织，父域可以代表总公司或总部，而子域可以对应不同的地理位置或分支机构。

子域

子域是从父域派生出来的子级域，通常用于表示组织结构中的不同部门、区域或功能单元。子域依赖于父域，但可以有自己独立的域控制器、资源、用户和安全策略。子域的创建通常是为了分隔不同的组织单位或管理区域，从而实现更加灵活和分散的管理。子域将组织内的资源和用户按照功能或部门划分，使得每个子域能够独立管理自己的计算机、用户和权限，子域可以继承父域的安全策略（如密码策略、账户锁定策略等），从而确保组织内统一的安全性。虽然子域继承父域的许多设置，但它仍然可以拥有自己的管理权限。子域可以独立管理用户、计算机、组策略等，并且有时可以自定义部分策略设置。

域树

建立信任关系的域集合，两个域相互之间可以访问，域树是由多个父域和子域组成的层次结构，通过它可以对组织内的计算机、用户和资源进行高效的管理。域树提供了分层次的管理和灵活的安全策略配置，同时确保跨域的资源访问与身份验证。它是大型组织和跨地域公司进行网络管理和资源分配的重要工具。

域森林

域森林是多个域树的集合，它是 Active Directory 中的最高层次的组织结构。在域森林中，所有的域都共享一个统一的目录服务架构和全局目录，但它们可以有不同的命名空间和安全边界。域森林提供了一个多域环境，在其中可以有多个父域和子域树，并且每个树可以有自己的命名空间。

域内计算机分类

域内的计算机可以根据其功能和用途进行不同的分类。域控制器是管理域中资源和用户的关键计算机，而成员计算机和服务器则提供不同类型的服务，如文件共享、应用托管、打印和网络基础设施服务等。通过这种分类，企业可以高效管理计算机、权限和资源，确保域内网络的安全性和高效性。

DNS域名服务器

在内网中，域名后缀通常使用内部域名（如 company.local 或 company.internal）来避免与公网域名冲突。在许多企业中，内网DNS与 Active Directory（AD） 紧密结合。域控制器（DC）通常是DNS服务器的关键组成部分。AD域内的所有资源（如计算机、用户、服务等）都由DNS进行名称解析。在AD环境中，DNS是一个重要的服务，支持域名注册和服务发现（例如，用户登录验证、文件共享等），员工可以使用便捷的域名访问公司内网资源，如共享文件夹、打印机、数据库等，而无需记住复杂的IP地址。当用户通过VPN或其他远程访问方式连接到内网时，内网DNS服务器帮助他们解析并访问内网资源，就像他们在局域网内一样。

域控制器

域控制器（Domain Controller，简称 DC）是一个负责管理网络域（Domain）的服务器，特别是在基于 Microsoft Active Directory（AD）的网络环境中。域控制器运行 Active Directory（AD） 服务，它提供一个目录服务来存储和管理网络资源的信息，它是组织内集中式管理用户、计算机、组和其他资源的核心组件，并提供身份验证和授权服务。域控制器允许管理员设置并管理访问权限、角色和策略。管理员可以通过 组策略（Group Policy）集中管理计算机和用户的配置，例如强制密码复杂性要求、安装软件、限制访问权限等，域控制器确保网络中的计算机和用户可以进行认证、访问权限控制和其他安全功能。

活动目录

概述

Active Directory（简称 AD）是由 Microsoft 提供的一个强大且灵活的目录服务，它用于管理企业内部网络中所有的计算机、用户、组和其他资源。Active Directory 主要应用于 Windows Server 操作系统环境中，是企业网络中最常见的集中式身份管理和资源访问控制解决方案。

功能

集中式管理：
Active Directory 提供一个中央位置，用于管理网络中的所有对象（如用户、计算机、打印机、文件夹、应用程序等），简化了网络资源的配置和维护。
身份验证和授权：
Active Directory 使用 Kerberos 或 NTLM 协议进行身份验证，确保只有授权用户才能访问网络资源。它还可以通过设置权限和组策略来控制用户访问不同资源的权限。
目录服务：
Active Directory 以树状结构（树形目录）存储和管理组织内的资源数据，包括用户、组、计算机、打印机等对象。它支持快速的查询操作，帮助管理员方便地查找和管理资源。
单点登录（SSO）：
使用 Active Directory，用户只需要一次登录，便可以访问同一域内的所有授权资源，而无需再次输入凭据。这提高了用户体验并简化了身份验证流程。
组织单位（OU）和组策略（GPO）：
Active Directory 允许将用户、计算机和其他资源按组织单位（OU）分类和分组。管理员可以根据组织的需求，定义具体的组策略（GPO）来集中管理计算机的设置（如密码策略、桌面环境设置等）。
跨域管理：
在大型企业中，Active Directory 支持跨多个域的管理。用户和资源可以在不同域之间共享，并且可以跨域进行身份验证。
全球目录（Global Catalog）：
Active Directory 支持一个全局目录服务器，它存储了跨整个域的所有对象的部分信息，支持快速查找和身份验证。

组件

域控制器（Domain Controllers，DC）：
域控制器是运行 Active Directory 的服务器，它负责身份验证、资源访问控制和复制 AD 数据。域控制器存储和管理 Active Directory 数据库中的信息，并通过复制确保多个 DC 上的数据一致性。
Active Directory 域（Domain）：
域是 Active Directory 中的基本管理单元，用于组织和管理用户、计算机等对象。每个域都有自己的数据库，存储关于该域的所有对象和设置。
组织单位（Organizational Units，OU）：
组织单位是 AD 中用于对对象进行逻辑分组的容器。它允许根据组织结构、部门或职能对对象进行分类，并且可以将组策略应用于 OU 中的所有对象。
目录服务协议（LDAP）：
LDAP（轻量级目录访问协议）是 Active Directory 用来进行查询和管理目录信息的协议。它允许客户端程序访问和管理 AD 中的对象。
全局编录服务器（Global Catalog）：
全局编录服务器存储所有域中的对象的部分属性，帮助加速跨域查询和身份验证。全局编录也可以作为一个域控制器角色来提供跨域的用户认证和资源查找服务。
组策略对象（GPO）：
组策略是 Active Directory 中的重要功能，允许管理员为域内的计算机和用户设置特定的安全策略、配置文件和其他操作系统设置。GPO 可以在整个域或组织单位（OU）上应用。

结构

域：
每个 AD 环境中的域是一个独立的命名空间，它包含用户、计算机和其他网络对象。在域内，所有的计算机和资源共享一套账户数据库。
域树（Domain Tree）：
域树由多个域组成，其中的每个域都有一个父域，并且在命名上是层级结构。例如，company.com 可以是父域，而 hr.company.com、it.company.com 则是子域。
域森林（Domain Forest）：
一个 域森林 是 Active Directory 中最顶层的结构，它包含一个或多个域树，并且域之间共享全局目录。域森林内的所有域都有共同的信任关系和统一的目录架构。

域本地组

概述

域本地组（Domain Local Groups，DLG）是 Active Directory（AD）中一种特殊的安全组类型，用于管理域内计算机和资源的访问权限。它是针对特定域内资源而创建的，旨在为该域内的资源提供精确的权限控制。不同于全局组（Global Groups），它通常被用来将组成员分配给 特定域的本地资源，例如文件夹、打印机、数据库等。域本地组只能授予访问该组所在域内的资源的权限，而不能跨域使用。它的权限是局限于域内的。域本地组的成员可以来自 任何域，不仅限于所在域中的用户、组或计算机。可以将来自不同域的用户、全局组、域本地组等添加到域本地组中。

权限

Administrators（管理员组） ————最重要的权限
Remote Desktop Users（远程登录组）

全局组

全局组（Global Group）是 Active Directory（AD）中的一种组类型，主要用于组织内用户、计算机和其他组的管理。全局组的作用通常是为了 管理和控制访问权限，并将组成员限制在特定的组织范围内。与 域本地组（Domain Local Group）不同，全局组的主要目标是对跨越多个域的用户和计算机进行分组。全局组的作用范围是 整个域内，它用于集中管理属于同一域的用户或计算机账户。这意味着全局组只能包含属于同一域的用户、计算机或其他组。全局组成员通常是按照职能或部门来分类的，例如财务组、技术组、市场组等。全局组的成员可以 跨域访问资源。例如，在一个跨多个域的环境中，虽然全局组的成员仅限于某个特定域，但它仍然可以被添加到其他域的域本地组中，从而获得访问这些域内资源的权限。

通用组

通用组（Universal Group）是 Active Directory（AD）中的另一种组类型，它与 全局组 和 域本地组 有所不同。通用组的主要作用是跨多个域提供灵活的访问控制，并能够集中管理跨越多个域的用户和资源权限。通用组的作用范围是整个 域森林，它可以包含来自 任何域 的用户、计算机、组（包括全局组和域本地组）。这使得它能够跨越多个域来管理权限，因此适用于需要跨域访问资源的场景。通用组的设计旨在服务于跨越多个域的环境，尤其适合大型企业中，涉及多个域和部门需要共享资源的情况。

AGDLP

AGDLP 是一种用于在 Active Directory 环境中进行访问控制和权限分配的安全模型，它由 A（Accounts）、G（Groups）、D（Domains）、L（Local Groups）和 P（Permissions）五个元素组成。该模型用于设计和实施有效的访问控制策略，确保资源的安全管理，并简化权限的管理。

元素

A (Accounts) - 用户账户：
用户账户是指企业网络中的个体用户。每个用户账户都被分配了一个唯一的标识符，通常是用户名和密码。用户账户属于一个特定的域，并可与其他组进行关联。
G (Groups) - 组：
组是 Active Directory 中的一种对象类型，它允许将多个用户账户归为一组，从而简化对权限的管理。组可分为两类：
安全组（Security Groups）：用于赋予权限和访问控制。
分发组（Distribution Groups）：用于电子邮件列表等，但不能赋予访问权限。
D (Domains) - 域：
域是 Active Directory 中的基本管理单元，它是网络资源（如用户、计算机等）的容器。每个域拥有自己的目录数据和安全策略。
L (Local Groups) - 本地组：
本地组是指域内计算机（工作站或服务器）上的本地组，用于控制对本地资源的访问。与域组不同，本地组只在计算机本地有效，不跨越域。
P (Permissions) - 权限：
权限是用户或组对计算机资源（如文件、文件夹、打印机、应用程序等）所拥有的访问权限。这些权限可以细分为读取、写入、执行等操作权限

模型

A 到 G（账户到组）：
用户账户应该属于域中的组，而这些组用于分配访问权限。每个用户账户都应被添加到一个或多个适当的安全组，这些安全组会决定该用户可以访问哪些资源。
G 到 D（组到域）：
将组应用于域级别的资源和权限控制。例如，用户组可以拥有对某个特定域中资源的访问权限。域管理员可以通过设置组策略，控制哪些组能访问域内的资源。
G 到 L（组到本地组）：
可以将域组添加到计算机的本地组中，从而使得域组的成员能够访问该计算机上的本地资源。例如，可以创建一个本地管理员组，将域管理员组添加进去，这样域管理员组的成员就能访问本地计算机的所有资源。
L 到 P（本地组到权限）：
本地组的成员拥有对本地计算机资源的访问权限。这些权限由计算机的管理员设置，允许用户执行特定的操作，如访问文件、打印机等。

Kerberos协议

概述

Kerberos协议是一个计算机网络认证协议，设计用来通过不安全的网络进行强身份验证。它使用了对称密钥加密和“票证”机制，主要用于确保通信双方的身份，并防止中间人攻击等安全问题。Kerberos协议广泛应用于各种分布式系统中，特别是在Windows环境中的Active Directory（AD）域环境中。

原理

Kerberos协议通过一个称为“票证”的机制来实现身份验证。其基本流程包括以下几个步骤：
初始化认证
用户通过客户端向认证服务器（AS，Authentication Server）请求身份验证。
客户端会发送一个带有用户名的请求，要求认证服务器为其颁发票证。
认证服务器会检查用户的身份，如果身份验证通过，AS会生成一个票证授权票证（TGT，Ticket Granting Ticket）。这个TGT被加密，只有TGT的服务器（通常是票证授权服务器，TGS）才能解密。
获取服务票证
一旦用户获得了TGT，用户就可以向票证授权服务器（TGS，Ticket Granting Server）请求访问某个特定服务的权限。
客户端使用TGT向TGS发送请求，并附带自己希望访问的服务名称。
如果TGT有效，TGS会生成一个新的服务票证，并将其返回给客户端。
访问目标服务
客户端使用服务票证与目标服务进行通信。服务票证包含了客户端的身份信息、加密密钥和会话密钥等数据。
目标服务验证票证的有效性后，允许客户端访问其资源。
会话密钥
每次服务的访问都会生成一个会话密钥，该密钥用于保护与目标服务的通信。只有客户端和目标服务能解密和加密会话中的数据。

组件

认证服务器（AS）： 负责验证用户的身份，并为其生成TGT。AS一般会从数据库中获取用户的密码或密钥。
票证授权服务器（TGS）： 在用户获得TGT后，TGS负责根据TGT生成并颁发服务票证。TGS是Kerberos协议中的重要部分，确保了不同服务的安全通信。
客户端： 发起身份验证请求，并使用获取到的票证与目标服务进行安全通信。
服务端： 各种提供资源的服务，通过验证客户端提交的服务票证来确认客户端的身份。

票据

票证是Kerberos协议的核心，它由以下几个部分组成：
TGT：票证授权票证，用户用来请求服务票证。TGT包含用户的身份信息和加密数据，由AS颁发。
服务票证：客户端用来访问特定服务的凭证。服务票证是由TGS生成并包含了目标服务的信息。
票证内容通常包括：
客户端标识。
服务标识。
票证有效期。
会话密钥（用于加密通信）。
